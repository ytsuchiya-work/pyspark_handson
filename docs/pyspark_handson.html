DATABRICKS · PYSPARK HANDS-ON

PySpark on Databricks ハンズオン資料

PySpark の概要、Databricks 上での実装方法、つまずきやすい注意点、そして合成データ生成から実行までを体験するハンズオン手順を 1 枚にまとめた資料です。

1. PySpark の概要

PySpark は、Python から Apache Spark を操作するための API です。学習・実装・保守がしやすい Python と、大規模分散処理エンジンである Spark の機能を組み合わせ、ビッグデータ処理と機械学習を実現します。Databricks は Apache Spark の上に構築されたプラットフォームであり、PySpark はその中核となるデータ処理インターフェースです。

習得が容易

柔軟で読みやすい Python を活用し、最小限の記述でデータ処理を表現できます。

豊富なデータ操作

列の選択・フィルター・結合・集計など、表形式データ向けの関数が充実しています。

スケーラビリティ

クラスター上で自動的に分散処理され、GB〜PB 級のデータを扱えます。

SQL との相互運用

DataFrame と Spark SQL を相互に行き来でき、チームの得意な言語で実装できます。

中核概念：DataFrame と遅延評価

  • DataFrame：名前付き列を持つ分散表形式データ。PySpark の基本データ構造です。
  • 変換（Transformation）：select / filter / join / groupBy など。新しい DataFrame を返すだけで、すぐには計算されません。
  • アクション（Action）：display / count / collect / write など。ここで初めて計算が走ります（遅延評価）。

2. Databricks 上での PySpark 実装

Databricks ノートブックでは spark（SparkSession）と dbutils が初期化済みで、すぐにコードを書き始められます。代表的な操作を以下に示します。

2.1 DataFrame の作成

# 値を直接指定して作成
df = spark.createDataFrame(
    data=[("Mikhail", 15), ("Zaky", 13), ("Zoya", 8)],
    schema=["name", "age"])

# Unity Catalog テーブルから作成
df_customer = spark.table("samples.tpch.customer")

# CSV ファイルから作成
df_csv = (spark.read.format("csv")
    .option("header", True)
    .option("inferSchema", True)
    .load(volume_file_path))

2.2 列・行の操作

from pyspark.sql.functions import col

# 列の選択 / 追加 / 型変換 / リネーム
df.select(col("c_custkey"), col("c_acctbal"))
df.withColumn("balance_flag", col("c_acctbal") > 1000)
df.withColumn("c_custkey", col("c_custkey").cast("string"))
df.withColumnRenamed("old_name", "new_name")

# フィルタ（複数条件は & / | を括弧で）
df.filter((col("c_nationkey") == 20) & (col("c_acctbal") > 1000))

# 重複削除 / NULL 処理 / ソート
df.distinct()
df.na.drop(); df.na.fill("0", subset=["c_acctbal"])
df.sort(col("c_acctbal").desc()).limit(10)

2.3 結合・集計・メソッドチェーン

from pyspark.sql.functions import avg, count

# 結合（inner / left / outer）
df_joined = df_order.join(df_customer,
    on=df_order["o_custkey"] == df_customer["c_custkey"], how="inner")

# 集計
df_customer.groupBy("c_mktsegment").agg(avg("c_acctbal"))

# メソッドチェーンで読みやすく
df_chained = (df_order
    .filter(col("o_orderstatus") == "F")
    .groupBy(col("o_orderpriority"))
    .agg(count(col("o_orderkey")).alias("n_orders"))
    .sort(col("n_orders").desc()))

2.4 SQL 連携・保存・データソース

# DataFrame ⇔ SQL
df_joined.createOrReplaceTempView("joined_view")
spark.sql("SELECT c_mktsegment, COUNT(*) FROM joined_view GROUP BY c_mktsegment")

# テーブルとして保存（Delta）
df_joined.write.mode("overwrite").saveAsTable(f"{catalog}.{schema}.result")

# 各種フォーマットで書き込み / 読み込み（Volume 上）
df.write.format("parquet").mode("overwrite").save(f"{volume}/out_parquet")
spark.read.format("json").load(f"{volume}/out_json")
カスタムデータソース： Databricks Runtime 15.4 LTS 以降（またはサーバレス環境バージョン 2）では、Python DataSource API で独自データソースの読み書きを実装できます（DataSource / DataSourceReader / DataSourceStreamReader など）。

3. 注意点・ベストプラクティス

カテゴリポイント
遅延評価変換は積み重ねても実行されない。count や display 等のアクションで初めて計算される。デバッグ時はこの違いを意識する。
大量データの収集collect() や toPandas() は全データをドライバーに集める。大規模データで実行するとメモリ不足になるため、件数を絞ってから使う。
行ごとのループ回避Python の for ループで 1 行ずつ処理せず、pyspark.sql.functions の列式で一括処理する（分散処理に乗せる）。
関数のインポートPython 組み込みの sum / max と衝突するため、from pyspark.sql.functions import sum as _sum のように別名にするか F. 接頭辞で使う。
UDF よりビルトインPython UDF はシリアライズのオーバーヘッドが大きい。まず組み込み関数や Spark SQL 関数で表現できないか検討する。
結合キーの曖昧さ両 DataFrame に同名列があると結合後に曖昧参照になる。事前に withColumnRenamed するか結合条件を明示する。
書き込みモードmode("overwrite") / "append" / "error"(既定) を意図して指定する。本番テーブルへの上書きは特に注意。
Unity Catalogテーブルは catalog.schema.table の 3 階層で参照する。ファイル入出力は DBFS ルートではなく /Volumes/...（ボリューム）を使うのが推奨。
ランタイム要件カスタムデータソースや一部関数は Databricks Runtime のバージョン依存。[UNSUPPORTED_FEATURE] が出たら 15.4 LTS 以降を選ぶ。

4. ハンズオン手順

付属のノートブック notebooks/01_pyspark_handson.py を Databricks 上で実行します。合成データの生成から各種 PySpark 操作までを順番に体験できます。

  1. Git フォルダの準備：Databricks のワークスペースで pyspark_handson リポジトリを Git フォルダとして追加（または既存フォルダを最新へ Pull）します。
  2. ノートブックを開く：notebooks/01_pyspark_handson.py を開きます。
  3. コンピュートをアタッチ：Serverless もしくは Databricks Runtime 15.4 LTS 以降の汎用クラスターに接続します。
  4. パラメータ設定：上部のウィジェットで出力先カタログ・スキーマ、生成件数を必要に応じて変更します。
  5. セットアップ実行：スキーマとボリュームが作成されます（セクション 0〜1）。
  6. 合成データ生成：顧客・注文データを生成し、Delta テーブルへ保存します（セクション 2〜5）。
  7. DataFrame 操作：列・行操作、結合、集計、メソッドチェーンを実行します（セクション 6〜10）。
  8. SQL 連携・データソース：一時ビュー経由の SQL 実行、CSV/JSON/Parquet の読み書きを確認します（セクション 11〜12）。
  9. 後片付け（任意）：最終セルのコメントを外すと、作成したスキーマを削除できます。
生成されるデータ： 既定で顧客 10,000 件・注文 50,000 件。num_customers / num_orders ウィジェットで増減できます。
